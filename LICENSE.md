## 🛠️ Troubleshooting Common Issues

**❌ Executor fails to attach / "Attach Failed" error**
This usually occurs after a Roblox patch before a compatibility update is released.
**Solution:** Check that `update_check` is set to `true` in `config.json` and relaunch the **Fluxus Z Executor** — a patch is typically available within a few hours of any major Roblox update.

**❌ Antivirus deletes the executor files on install**
Security software commonly flags injection tools as threats due to the nature of their memory access.
**Solution:** Before installing, whitelist `C:/Program Files/FluxusZ/` in your antivirus exclusion settings, then re-download and reinstall cleanly.

**❌ Android `.apk` install blocked — "App not installed" error**
Your device has not enabled sideloading permissions, or the `.apk` downloaded incompletely.
**Solution:** Go to **Settings → Security → Install Unknown Apps** and enable it for your browser or file manager, then re-download the `.apk` and retry.

**❌ iOS `.ipa` showing "Untrusted Developer" after install**
This is standard iOS behavior for sideloaded apps not distributed via the App Store.
**Solution:** Go to **Settings → General → VPN & Device Management**, find the developer profile associated with the `.ipa`, and tap **Trust**.

**❌ Script executes but has no visible effect in-game**
The script may require specific game conditions, or it ran before the game's UI finished loading.
**Solution:** Increase `injection_delay_ms` to `1500` in `config.json` and try re-executing after the game environment is fully loaded.

**❌ High CPU usage while executor is running in background**
The default mode keeps active threads running for fast attach response.
**Solution:** Enable `low_power_mode: true` in `config.json` to drop background CPU usage significantly without impacting script performance.
